Quelle est l'utilité de l'attribut "alt" de la balise "img" ?

L'attribut alt de la balise img est une description textuelle globale de l'image. Il est principalement utile dans deux cas :

  • Dans le cas où il y a une erreur avec la récupération de l'image et celle-ci ne s'affiche pas.
  • Dans le cas où des personnes ayant des difficultés visuelles telles que des malvoyants accèdent à l'image. Ceux-ci pourront avoir une description de l'image faute de la voir. Un logiciel appelé lecteur d'écran servira à lire le contenu de l'attribut alt (qui est invisible de base).

Quelle est la différence entre une balise "span" et une balise "div" ?

La différence entre une balise "span" et une balise "div" est la suivante :
Une balise "span" est une balise de type inline. Elle utilise la place de son contenu et ne commence pas sur une nouvelle ligne. Quant à la balise "div" (division), c'est une balise de type bloc. En hauteur, elle utilise la taille de son contenu, en revanche, en largeur, elle utilise 100% de la largeur disponible de son conteneur parent et commence sur une nouvelle ligne par défaut. Comment peut-on surligner du texte en HTML ?

Pour surligner une partie d'un texte en HTML, il suffit de la placer dans une balise "mark".

Je suis un texte et cette partie est surlignée.

Cette balise n'apporte pas de valeur sémantique au texte. Il est préférable d'utiliser du CSS pour le style y compris pour le surlignage. Cela permettra d'avoir un fichier HTML plus simple et donnera la possibilité de choisir la couleur de surlignage.

Quels sont les points clés et les objectifs d'HTML5

HTML5 a été créé pour remplacer HTML4, XHTML et HTML DOM niveau 2.

HTML5 fournit de nombreux avantages, dont les principaux sont listés ci-dessous

  • Ajouter des médias tels que des vidéos, des graphiques, etc., sans avoir besoin de passer par des plugins tels que Flash
  • HTML5 fournit un meilleur support sémantique pour la structure des pages web grâce à de nouvelles balises structurelles.
  • Établir une norme d'analyse plus stricte afin d'homogénéiser l'interprétation des fichiers HTML par les navigateurs.
  • Simplification de la gestion des erreurs grâce à des règles plus claires concernant la syntaxe HTML.
  • Compatibilité avec les documents HTML plus anciens comme HTML4 et XHTML. Cela permet une mise à jour progressive vers HTML5 sans avoir à recoder tout le site.
  • Une meilleure adaptabilité entre les différentes tailles d'écran selon les appareils.
  • Amélioration de la compatibilité des navigateurs mobiles en lien notamment avec les écrans tactiles et les interactions mobiles.
  • HTML5 met l'accent sur des performances accrues.

Qu'est-ce qu'un encodage de caractères ?

Pour afficher correctement une page HTML, un navigateur a besoin de savoir quel est l'encodage de caractères à utiliser pour lire le fichier. Cet encodage doit être spécifié dans une balise "meta" à l'intérieur de la balise head du fichier HTML. Voici un exemple:

HTML4 :

            <meta http-equiv="Content-Type" content="text/html;charset=ISO-8859-1"/>

HTML5 :

            <meta charset="utf-8"/>

Qu'est-ce qu'une balise auto-fermante ?

Une balise auto-fermante est une balise unique ne fonctionnant pas par paire comme la plupart des balises HTML. Voici plusieurs exemples de balises fermantes :

  • Saut de ligne:
    <br/>
  • Input
    <input type="text" />
  • Image :
    <img width="200" alt="Un chiot marron et blanc" src="https://loreandscience.fr/wp-content/uploads/2022/06/blog-croissance-chiot-lore.webp" />

Certaines balises n'ont pas spécialement besoin d'un anti-slash fermant /. Cette balise est tout à fait valide : <meta charset="utf-8">

Dans le doute, il est tout de même préférable d'utiliser un anti-slash fermant pour chaque balise auto-fermante.

Quelle est la différence entre un attribut et une propriété en HTML ?

Un attribut est inscrit en dur directement dans l'élément HTML; quant à une propriété, elle existe sans être inscrite en dur. On peut y accéder et la modifier en utilisant JavaScript ou CSS.

En HTML, qu'est-ce que "Le mode de compatibilité", le "mode presque standard", et le "mode standard complet" ?

Mode de compatibilité ou mode de non-conformité (Quirks mode):

C'est un mode qui est activé par le navigateur lorsque la page HTML ne suit pas strictement les normes et les standards du W3C (World Wide Web Consortium) ou lorsque la déclaration du doctype indique une ancienne version d'HTML.

Mode presque standard (Quirks mode limited):

C'est un mode activé par le navigateur qui tente de retranscrire le code HTML rencontré. Ce code n'est pas totalement aux normes ou bien il est écrit via une version HTML plus ancienne mais assez récente pour ne pas avoir à activer le mode Quirks et rester en mode Quirks mode limited.

Mode standard complet (Full standard mode):

Ce mode est activé lorsque le code HTML d'une page est conforme aux normes du World Wide Web Consortium (W3C). Dans ce mode, le navigateur traduit strictement le code HTML, CSS et JavaScript, ce qui garantit une meilleure uniformité des sites web entre les différents navigateurs.

Quelle est la différence entre un élément inline et un élément bloc ?

Élément inline : Les éléments inline utilisent l'espace de leur contenu en hauteur et en largeur. Ils s'intègrent à la ligne où ils sont appelés et n'ont pas besoin d'une nouvelle ligne pour eux.

Éléments block : Les éléments block sont des éléments HTML utilisant 100% de la largeur disponible de leur parent le plus proche. Chaque nouvel élément block commence sur une nouvelle ligne par défaut.

Qu'est-ce qu'une balise optionnelle en HTML ?

En HTML, une balise optionnelle est une balise qui n'est pas nécessaire pour que le code HTML soit valide.

Par exemple, les balises "meta", "head" et "li" dans des listes ordonnées ou non, sont des balises optionnelles.

Malgré tout, il est recommandé d'utiliser ces balises pour une meilleure lisibilité du code HTML par les autres développeurs ou vous-même.

Qu'est-ce que l'attribut defer en HTML ?

En HTML, l'attribut defer sur une balise script permet d'attendre la fin du chargement de la page HTML pour ensuite commencer à charger le fichier JavaScript. Cela améliore la performance et ne bloque pas l'affichage de la page HTML. Defer = Déférer / reporter

Que signifie l'attribut async sur une balise script en HTML ?

En HTML, l'attribut async signifie que le chargement de la page et du fichier HTML se font en même temps. Une fois le fichier JavaScript chargé, il est exécuté.

Comment gérer plusieurs langues pour une page HTML ?

Pour gérer plusieurs langues pour une page HTML, plusieurs actions sont nécessaires. Tout d'abord, il faut renseigner la langue principale dans la balise HTML comme suit : <html lang="fr"></html>

Ensuite, pour les textes en différentes langues, vous devez rédiger un texte pour chaque langue que vous désirez intégrer à votre page web. Ensuite, grâce à l'attribut lang, vous pourrez spécifier quelle langue appartient à quel texte. Exemple :

<p lang="fr">Ceci est un texte en français</p>

<p lang="en">This is an English text</p>

Ensuite, selon la langue sélectionnée par défaut dans le navigateur par l'utilisateur, celui-ci affichera tel ou tel texte.

Qu'y a-t-il de nouveau dans HTML5 ?

Dans HTML5, les nouveautés sont :

  • Une meilleure sémantique grâce à l'ajout de nouvelles balises à valeur sémantique.
  • Une prise en charge de certains médias tels que les vidéos ou les graphiques qui ne nécessite désormais plus le besoin d'avoir un plugin tel que Flash installé sur son ordinateur.
  • Des normes plus strictes permettant une meilleure standardisation entre navigateurs.
  • HTML5 permet un stockage local sur l'ordinateur de l'utilisateur.
  • HTML5 améliore les performances grâce à la prise en charge de tâches asynchrones et également la prise en charge de la géolocalisation.

Quelle est la différence entre la balise "div" et la balise "section" ?

La balise div sert à diviser la page web en plusieurs parties. Elle n'apporte aucune valeur sémantique et son contenu peut être aléatoire.

La balise "section" est une balise permettant de sectionner la page web en parties abordant le même thème comme "contact", "qui sommes-nous", "accueil" ou parlant d'un sujet du même thème tel qu'Halloween.

À quoi sert l'attribut rel="noopener" ?

L'attribut noopener est très utile en termes de sécurité. En effet, lorsqu'un lien est ouvert dans une nouvelle fenêtre via l'attribut target="_blank", la nouvelle fenêtre est encore en lien avec la fenêtre parente. Des attaques peuvent venir via ce lien/ouverture. L'attribut rel="noopener" permet de couper cette ouverture vers la fenêtre parente. Il évite ainsi les risques liés à cette attaque.

Une page HTML peut-elle contenir plusieurs headers ou plusieurs footers ?

Une page HTML peut contenir, pour le contenu de la page globale (body), un seul header et un seul footer. Néanmoins, une section ou un article peuvent également comporter un header et un footer. En résumé, dans une page HTML, il peut y avoir plusieurs balises header et footer, mais le document HTML doit quant à lui n'avoir qu'un seul header et un seul footer.

Qu'est-ce que WebSQL ?

WebSQL est (était) une base de données intégrée au navigateur de l'utilisateur. Il n'est plus dans les normes du W3C aujourd'hui.

Qu'est-ce que le DOM ?

Le DOM (Document Object Model) est l'objet modélisé du document HTML.

Lorsque l'on interagit avec les balises HTML, les attributs, etc., on n'interagit pas sur le code en dur dans le fichier .html, mais plutôt sur sa représentation numérique, le DOM.

HTML5 a introduit la notion de localStorage et sessionStorage, expliquez-les

Grâce à HTML5, il est possible de stocker des informations directement dans le navigateur côté client. Contrairement aux cookies qui sont stockés côté client mais envoyés au serveur pendant les requêtes HTTP, localStorage et sessionStorage stockent les données uniquement côté client. La différence avec les cookies se trouve également au niveau de l'espace disponible pour stocker les données. L'espace pour stocker les données des cookies étant bien plus limité que celui des deux méthodes évoquées précédemment. La troisième différence réside dans la durée de vie des données. La différence entre localStorage et sessionStorage réside dans la durée de vie des données et leur portée. localStorage conserve les données même après la fermeture du navigateur et partage celles-ci avec toutes les fenêtres du même navigateur. sessionStorage, quant à lui, conserve les données uniquement sur une fenêtre et les supprime une fois la fenêtre ou le navigateur fermé.

Quelle est la différence entre XHTML et HTML ?

XHTML est également un langage de balisage, mais il est plus strict dans sa syntaxe qu'HTML. Néanmoins, avec l'arrivée d'HTML5, HTML se rapproche un peu plus des standards d'XHTML.

Qu'est-ce que WebSQL ?

C'est une API de base de données locale servant à stocker des données dans le navigateur. Elle est aujourd'hui obsolète et il est fortement déconseillé de l'utiliser.

Qu'est-ce que le cache busting ?

Le cache busting est une méthode permettant d'éviter que les navigateurs gardent des fichiers obsolètes en cache au lieu de les mettre à jour. Plusieurs techniques permettent d'effectuer ce "cache busting".

  • Changer le nom du fichier mis à jour.
  • Dans les en-têtes HTTP, il est possible de mettre une date d'expiration pour le cache.
  • Utiliser un outil de construction comme Webpack.

Qu'est-ce que la spécification HTML et l'implémentation du navigateur ?

La spécification HTML est la documentation officielle établie par le W3C (World Wide Web Consortium). Elle établit les règles et la syntaxe HTML. Elle est évolutive.

L'implémentation du navigateur est la façon dont les navigateurs vont retranscrire cette spécification HTML. Les interprétations peuvent varier et certains navigateurs implémentent de nouvelles fonctionnalités ou sont en retard sur d'autres, ce qui crée des incohérences entre une même page web sur différents navigateurs.

À quoi sert le DOCTYPE ?

Il sert au navigateur à connaître le type du document, ce qui permet de meilleures performances et une meilleure interprétation du code, et évite également les erreurs d'interprétation. Le DOCTYPE est souvent une version HTML ou XHTML.

Qu'est-ce que le "progressive rendering" (rendu/affichage progressif) en HTML ?

Le rendu progressif en HTML consiste à améliorer l'expérience utilisateur en affichant le plus rapidement possible les éléments susceptibles d'apparaître en premier pour l'utilisateur, tels que le header (d'où l'importance de la balise header), le texte (sans forcément charger sa police d'écriture à ce moment-là), les fichiers HTML, CSS et JavaScript. Le tout se fait de manière asynchrone et dès qu'une ressource est chargée, elle apparaît sur le site web.

À quoi sert l'attribut srcset de la balise img ?

Il permet de proposer au navigateur plusieurs images en donnant une courte description de celles-ci, et le navigateur choisira automatiquement l'image à télécharger selon la taille de l'écran de l'utilisateur. Cela permet une économie de la bande passante et donc accroît les performances.

Quel est le rôle de la balise "main" en HTML

La balise "main" en HTML est utilisée pour regrouper le contenu principal d'une page web. Il ne doit y avoir qu'une seule balise "main" par page HTML. La balise main a une valeur sémantique pour les navigateurs. La balise main facilite la navigation pour les lecteurs d'écran et donc l'accessibilité pour les personnes réduites. La balise main est de type bloc.

Pourquoi faut-il utiliser les balises sémantiques dans HTML ?

Il est très fortement recommandé d'utiliser des balises sémantiques dès que cela est possible dans un fichier HTML. Les balises sémantiques améliorent fortement le référencement naturel des pages web.

Qu'est-ce que le format "webp" ?

Le format webp est un format d'image développé par Google spécialement conçu pour le web. Il offre de nombreux avantages comme des performances accrues grâce à sa taille, une excellente qualité d'image, et une forte compatibilité avec les navigateurs.

Peut-on générer une clé publique en HTML ?

Il est techniquement possible de générer une clé publique dans un fichier HTML via les balises "script". Néanmoins, il est très fortement déconseillé d'utiliser cette méthode pour générer une clé publique pour des raisons de sécurité.

Pourquoi est-il plus judicieux de placer le fichier CSS avant la balise "script" dans un fichier HTML ?

Il est plus judicieux de placer un fichier CSS avant la balise "script" pour plusieurs raisons :

  • Le chargement de la structure HTML ainsi que de son design se feront avant le chargement du fichier script, qui peut être volumineux et lent à charger.
  • En cas de problème avec le script, la page web affichera tout de même une interface utilisateur agréable grâce au CSS chargé avant le script.

Qu'est-ce qu'un composant web ?

Un composant web, également appelé widget, est un bloc de code réutilisable. Un composant web peut normalement fonctionner indépendamment et possède sa propre logique.
