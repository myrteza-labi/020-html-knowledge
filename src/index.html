Une iframe est un conteneur dans lequel on intègre une page web externe.

Quelle est l'utilité de l'attribut "alt" de la balise "img" ?

L'attribut alt de la balise img est une description textuelle globale de l'image. Il est principalement utile dans deux cas :

  • Dans le cas où il y a une erreur avec la récupération de l'image et celle-ci ne s'affiche pas.
  • Dans le cas où des personnes ayant des difficultés visuelles telles que des malvoyants accèdent à l'image. Ceux-ci pourront avoir une description de l'image faute de la voir. Un logiciel appelé lecteur d'écran servira à lire le contenu de l'attribut alt (qui est invisible de base).

Quelle est la différence entre une balise "span" et une balise "div" ?

La différence entre une balise "span" et une balise "div" est la suivante :
Une balise "span" est une balise de type inline. Elle utilise la place de son contenu et ne commence pas sur une nouvelle ligne. Quant à la balise "div" (division), c'est une balise de type bloc. En hauteur, elle utilise la taille de son contenu, en revanche, en largeur, elle utilise 100% de la largeur disponible de son conteneur parent et commence sur une nouvelle ligne par défaut. Comment peut-on surligner du texte en HTML ?

Pour surligner une partie d'un texte en HTML, il suffit de la placer dans une balise "mark".

Je suis un texte et cette partie est surlignée.

Cette balise n'apporte pas de valeur sémantique au texte. Il est préférable d'utiliser du CSS pour le style y compris pour le surlignage. Cela permettra d'avoir un fichier HTML plus simple et donnera la possibilité de choisir la couleur de surlignage.

Quels sont les points clés et les objectifs d'HTML5

HTML5 a été créé pour remplacer HTML4, XHTML et HTML DOM niveau 2.

HTML5 fournit de nombreux avantages, dont les principaux sont listés ci-dessous

  • Ajouter des médias tels que des vidéos, des graphiques, etc., sans avoir besoin de passer par des plugins tels que Flash
  • HTML5 fournit un meilleur support sémantique pour la structure des pages web grâce à de nouvelles balises structurelles.
  • Établir une norme d'analyse plus stricte afin d'homogénéiser l'interprétation des fichiers HTML par les navigateurs.
  • Simplification de la gestion des erreurs grâce à des règles plus claires concernant la syntaxe HTML.
  • Compatibilité avec les documents HTML plus anciens comme HTML4 et XHTML. Cela permet une mise à jour progressive vers HTML5 sans avoir à recoder tout le site.
  • Une meilleure adaptabilité entre les différentes tailles d'écran selon les appareils.
  • Amélioration de la compatibilité des navigateurs mobiles en lien notamment avec les écrans tactiles et les interactions mobiles.
  • HTML5 met l'accent sur des performances accrues.

Qu'est-ce qu'un encodage de caractères ?

Pour afficher correctement une page HTML, un navigateur a besoin de savoir quel est l'encodage de caractères à utiliser pour lire le fichier. Cet encodage doit être spécifié dans une balise "meta" à l'intérieur de la balise head du fichier HTML. Voici un exemple:

HTML4 :

            <meta http-equiv="Content-Type" content="text/html;charset=ISO-8859-1"/>

HTML5 :

            <meta charset="utf-8"/>

Qu'est-ce qu'une balise auto-fermante ?

Une balise auto-fermante est une balise unique ne fonctionnant pas par paire comme la plupart des balises HTML. Voici plusieurs exemples de balises fermantes :

  • Saut de ligne:
    <br/>
  • Input
    <input type="text" />
  • Image :
    <img width="200" alt="Un chiot marron et blanc" src="https://loreandscience.fr/wp-content/uploads/2022/06/blog-croissance-chiot-lore.webp" />

Certaines balises n'ont pas spécialement besoin d'un anti-slash fermant /. Cette balise est tout à fait valide : <meta charset="utf-8">

Dans le doute, il est tout de même préférable d'utiliser un anti-slash fermant pour chaque balise auto-fermante.

Quelle est la différence entre un attribut et une propriété en HTML ?

Un attribut est inscrit en dur directement dans l'élément HTML; quant à une propriété, elle existe sans être inscrite en dur. On peut y accéder et la modifier en utilisant JavaScript ou CSS.

En HTML, qu'est-ce que "Le mode de compatibilité", le "mode presque standard", et le "mode standard complet" ?

Mode de compatibilité ou mode de non-conformité (Quirks mode):

C'est un mode qui est activé par le navigateur lorsque la page HTML ne suit pas strictement les normes et les standards du W3C (World Wide Web Consortium) ou lorsque la déclaration du doctype indique une ancienne version d'HTML.

Mode presque standard (Quirks mode limited):

C'est un mode activé par le navigateur qui tente de retranscrire le code HTML rencontré. Ce code n'est pas totalement aux normes ou bien il est écrit via une version HTML plus ancienne mais assez récente pour ne pas avoir à activer le mode Quirks et rester en mode Quirks mode limited.

Mode standard complet (Full standard mode):

Ce mode est activé lorsque le code HTML d'une page est conforme aux normes du World Wide Web Consortium (W3C). Dans ce mode, le navigateur traduit strictement le code HTML, CSS et JavaScript, ce qui garantit une meilleure uniformité des sites web entre les différents navigateurs.

Quelle est la différence entre un élément inline et un élément bloc ?

Élément inline : Les éléments inline utilisent l'espace de leur contenu en hauteur et en largeur. Ils s'intègrent à la ligne où ils sont appelés et n'ont pas besoin d'une nouvelle ligne pour eux.

Éléments block : Les éléments block sont des éléments HTML utilisant 100% de la largeur disponible de leur parent le plus proche. Chaque nouvel élément block commence sur une nouvelle ligne par défaut.

Comment définir le mode de compatibilité d'Internet Explorer en HTML

Pour activer le mode de compatibilité d'Internet Explorer en HTML, il faut intégrer la balise meta suivante : <meta http-equiv="X-UA-Compatible" content="IE=11" />

Voici les valeurs possibles pour l'attribut X-UA-Compatible :

  • "IE=edge"
  • "IE=11"
  • "IE=EmulateIE11"
  • "IE=10"
  • "IE=EmulateIE10"
  • "IE=9"
  • "IE=EmulateIE9"
  • "IE=8"
  • "IE=EmulateIE8"
  • "IE=7"
  • "IE=EmulateIE7"
  • "IE=5"

Qu'est-ce qu'une balise optionnelle en HTML ?

En HTML, une balise optionnelle est une balise qui n'est pas nécessaire pour que le code HTML soit valide.

Par exemple, les balises "meta", "head", et "li" dans des listes ordonnées ou non, sont des balises optionnelles.

Néanmoins, il est recommandé d'utiliser ces balises pour une meilleure lisibilité du code HTML par les autres développeurs ou vous-même.

Comment change-t-on la direction d'un texte dans un fichier HTML ?

Il existe plusieurs façons de changer la direction du texte en HTML.

Élément par élément :

<p dir="ltr">Ce texte se lit de gauche à droite </p>

Ce texte se lit de gauche à droite

<p dir="rtl">Ce texte se lit de droite à gauche <p>

Ce texte se lit de droite à gauche

Ce changement de direction peut également se faire de manière générale dans la valeur de l'attribut "dir" de la balise html, par exemple :

<html dir="rtl">

Qu'est-ce que l'attribut "defer" en HTML ?

En HTML, l'attribut "defer" sur une balise <script> permet d'attendre la fin du chargement de la page HTML pour ensuite commencer à charger le fichier JavaScript. Cela améliore les performances et ne bloque pas l'affichage de la page HTML.

Que signifie l'attribut async sur une balise <script> en HTML ?

En HTML, l'attribut async signifie que le chargement de la page et du fichier HTML se font en même temps. Une fois le fichier JavaScript chargé, il est exécuté.

Qu'est-ce que la balise <small> en HTML ?

La balise <small> est une balise indiquant que son contenu est de moindre importance.

Exemple :

L'ours brun du Canada est un ours pouvant peser plus de 500 kilos. Il aime manger du miel.

Comment gérer plusieurs langues pour une page HTML ?

Pour gérer plusieurs langues pour une page HTML, plusieurs actions sont nécessaires.

Tout d'abord, il faut renseigner la langue principale dans la balise HTML, par exemple :

<html lang="fr">

Ensuite, pour les textes en différentes langues, vous devez rédiger un texte pour chaque langue que vous désirez intégrer à votre page web. Ensuite, grâce à l'attribut lang, vous pourrez spécifier quelle texte appartient à quelle langue. Exemple :

<p lang="fr">Ceci est un texte en français</p> <p lang="en" >This is an English text</p>

Ensuite, selon la langue sélectionnée par défaut dans le navigateur par l'utilisateur, celui-ci affichera tel ou tel texte.

Qu'est-ce qui est nouveau dans HTML5 ?

Dans HTML5, les nouveautés sont :

  • Une meilleure sémantique grâce à l'ajout de nouvelles balises à valeur sémantique.
  • Une prise en charge de certains médias tels que les vidéos ou les graphiques qui ne nécessitent désormais plus le besoin d'avoir un plugin tel que Flash installé sur son ordinateur.
  • Des normes plus strictes permettant une meilleure standardisation entre navigateurs.
  • HTML5 permet un stockage local sur l'ordinateur de l'utilisateur.
  • HTML5 améliore les performances grâce à la prise en charge de tâches asynchrones et également la prise en charge de la géolocalisation.

Quelle est la différence entre la balise "div" et la balise "section" ?

La balise <div> sert à diviser la page web en plusieurs parties. Elle n'apporte aucune valeur sémantique et son contenu peut être aléatoire.

La balise <section> est une balise permettant de sectionner la page web en parties abordant le même thème comme "contact", "qui sommes-nous", "accueil" ou parlant d'un sujet du même thème tel qu'Halloween.
